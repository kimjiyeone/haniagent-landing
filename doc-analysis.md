# 사업계획서 분석 (p6-10)

## 핵심 기능 3대 카테고리 (p7 - 매우 중요한 기능 소개 이미지들!)

### 1. 진료 자동화
- **AI 음성 차팅**: "진료 중 녹음만 하세요. AI가 SOAP 차트를 만듭니다."
  - 음성 → 텍스트 자동 변환, 한의학 SOAP 형식 차트 자동 생성, 확정 1번으로 완료
- **EMR 입력도 복사 → 붙여넣기 1초**: 확정된 SOAP 차트가 EMR 파일 대기열에 자동 생성
  - 자동 대기열 정렬, 어떤 EMR에도 호환, 이중 입력 제거

### 2. 환자/직원 관리
- **실장은 인박스만 따라하면 됩니다**: 원장이 확정한 차트에서 태스크가 자동 생성, 우선순위 정렬, 신입 Day 1부터 업무 가능
- **차트 기반 맞춤 카톡, AI가 자동 생성합니다**: T1 예약확인, T2 D-1 리마인드, T3 D+1 사후관리
  - 차트 기반 AI 맞춤 메시지, 추후 자동 발송 업그레이드 예정

### 3. 경영 인사이트
- **재진율, 매출, 환자 현황 숫자로 한눈에**: 재진율/매출 실시간 추적, 환자 유형별 분포 분석, 데이터 기반 경영 의사결정
- **환자 유형을 AI가 분석하고, 맞춤 소통 팁을 제안합니다**: 환자 유형 자동 분류, 맞춤 치료 접근법 제안, 환자 만족도 분석

# 기획서 분석 (핵심 요약)

## 뾰족한 문제
"직원 역량에 의존하는 한의원 운영" → 직원이 바뀌면 환자 관리 품질 저하, 원장이 일일이 교육/점검, 누락 발생 → 환자 이탈

## 솔루션 한 문장
"원장이 말한 다음 방문 시점을, 시스템이 자동으로 챙겨서, 직원이 바뀌어도 환자가 다시 오게 만든다."

## 코어 루프 (상세)
원장 녹음 → AI SOAP 자동생성 → 원장 [확정] → 권장시점 1탭 → 시스템 T1 자동생성 → D-1에 실장 인박스에 T2 → 실장 [카톡 발송] → 환자 재방문 → 실장 체크인 → 태스크 자동 해소 → 루프 반복

## 원장이 하는 일 (3가지만)
1. 녹음 시작/종료 (버튼 2번)
2. [확정] — SOAP 확인 후 1탭
3. 권장시점 탭 — AI 추천값 확인/변경 (1탭)

## 실장이 하는 일 (인박스만)
1. 체크인 — 이름 검색 + 탭
2. 태스크 처리 — 인박스 순서대로
3. 카톡 발송 — [발송] 1탭

## 태스크 체인
- T1: 예약관리 (확정 시 자동생성) — 예약 안내 카톡
- T2: 1차 리마인드 (권장일 D-1 cron) — 리마인드 카톡
- T3: 2차 리마인드 (권장일 당일 미체크인) — 2차 리마인드 카톡
- T4: 리핏 (원장 리핏 버튼) — EMR 리핏 처리

## 카톡 템플릿 예시
- T1: "안녕하세요 {name}님, {clinic}입니다. 원장님께서 {name}님의 {chief_complaint} 치료 경과를 위해 {date}경 내원을 권해 주셨어요. 예약 도와드릴까요?"
- T2: "{name}님, {clinic}입니다. 내일 {date} 진료 예정이세요. 내원 가능하시면 답변 부탁드려요 :)"
- T3: "{name}님, {clinic}입니다. 오늘 진료 예정이셨는데 괜찮으세요? 일정 변경이 필요하시면 연락주세요."
